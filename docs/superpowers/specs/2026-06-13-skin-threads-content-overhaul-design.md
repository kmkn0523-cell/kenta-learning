## 2. 5視点診断のサマリー

### ① フック力（1行目）
- 73%（93/128）の1行目が丁寧語の講義調 → フィードで「広告/bot」に見えてスキップされる
- 「原因ファースト病」: 塩分・血糖値・ビタミン等、読者の脳内に無い単語から始めて自分事化ゼロ
- 勝ち型「〜してるのに結果が出ない（努力の裏切り）」が量産部隊の tree に未移植
- **原則**: 1行目は症状・場面から / 25字以内・タメ口・否定形や体感語で止める / 研究引用と資料集の数字は1行目禁止（深掘りへ）/ 教科書ワードは症状チェックで好奇心ギャップを作る

### ② 保存価値
- 深掘り(posts[1])がメカニズム解説中心で、保存して使う「道具」が無い/後ろに埋もれる
- リスト項目に数値（温度・回数・分量）が無く、手順書として未完成
- **保存資産が、リーチの低い1コメ目（返信側）に置かれている**（親で予告もしない）
- **原則**: 保存物は親（高リーチ側）に置く / 全項目に数字 / 1投稿1用途（買い物・やめる・◎×・週次・持ち物）/ 利用シーンを明示（「コンビニで開いて」「生理3日前に」）/ 共通5習慣を主役にせず固有項目3つ以上 / スクショ1枚15行以内

### ③ 拡散・リポスト力
- リポスト導線が128テーマ中0件。CTAが全て「保存/コメント/note」の私的アクション
- 全投稿が当事者向け教材 → リポスト＝「私は肌が汚い」の自己開示になるスティグマ構造（第三者宛てが0）
- スクショ・引用に耐える「断言の1行」が無く、情報が【】①②③に均等に薄まっている
- **原則**: 1行目に読者のセリフ / スクショ単体で成立する断言を1行・空行で独立配置 / 1本につきヴィラン（高い化粧水神話・毛穴パック等）を1つ斬る / 週2本は「第三者に送る口実」型（1投稿1アクション）/ ヘッジは1回まで・具体数字＋身近な比較に

### ④ フォロー転換力
- 注目が集まる深掘り113/128が「顔のない百科事典」。役立つが「人」がいないのでフォローされない
- 唯一の資産「8年の物語」が枕詞化（26テーマで同じ「8年間〜」、章立てエピソード0）
- 連載性・次回予告が128中0件。一人称の感情語は4テーマのみ
- **原則**: 「8年」を章立て台帳（1年目皮膚科3院/3年目化粧水3万円/5年目腸活/7年目引き算）にして1投稿1章 / 深掘り冒頭に「失敗→転機→期間つき変化」/ 毎投稿に連載フック / 1投稿に1つ安全な一人称の感情（鏡を見たくなかった等）

### ⑤ Threadsアルゴリズム適合（2026）
- 伸びはhuman_postなのに主力がtree（形式選択がデータと逆行）
- 【見出し】①②③のnote目次体はリプライを生まず初速で落とされる
- タグ3〜5個自動付与・フォロー/リポスト懇願はエンゲージメントベイト降格対象
- 親が「予告だけ・中身ゼロ」のティーザーで親単体のいいね/滞在を取れない
- **原則**: human_post（完結型）をデフォルトに / 会話体統一（見出し番号排除・です/ます2文以内・1文40字）/ タグ1個・テーマ固有 / CTAは「思い当たる人は保存を」中心、フォロー/リポスト依頼文は削除 / 末尾の問いかけは半分以下・Yes/No共感型

## 3. 新規投稿パックの設計

全24本。診断⑤に従い **全件 human_post（完結型・skip_image=true）**。保存資産を親（高リーチ側）に置き、タグは本文末尾1個に内蔵。

| パック | id | 本数 | 狙いと型 |
|---|---|---|---|
| 保存特化 | 201-206 | 6 | 親に【保存用】リストを内蔵。場面起点（コンビニ/生理前/外食/旅行/逆効果/緊急リセット）・全項目数字・利用シーン明示・締めは「○○のとき用に保存を」 |
| 拡散特化 | 207-212 | 6 | スクショ単体で成立する断言＋ヴィラン1体＋第三者宛て。化粧水神話/毛穴パック/洗いすぎを斬り、共感の代弁（恥の言語化）を1本含む |
| フォロー特化 | 213-218 | 6 | 「8年の記録①〜⑥」連載。台帳に沿った章立て・一人称の感情・毎話末に連載フック。⑤はマニフェスト（固定ポスト級）、⑥は30日チャレンジ予告で参加導線 |
| ボーナス弾 | bonus 5-10 | 6 | 反応good日の追撃用・完結型単発。一撃名言＋Yes/No参加型。既存弾4本（残弾0消化）を補充 |

### 台帳（人格の一貫性のため固定）
高校2年で発症 → 1年目:皮膚科3院 / 3年目:3万円の化粧水（変わらず）/ 5年目:腸活 / 7年目:「足す」→「引く」転換（寝る前カフェラテをやめ23時就寝）/ 8年目(今):落ち着いた。化粧品に総額「数十万円」。**別投稿でも同じ事実・同じ数字を使う**。

## 4. 自己検証結果

- **薬機法**: 違反0件。「効く/治る/治す/○○%効果」は活用形含めゼロ。検出された「効果」3件はすべて「逆効果」（既存検証済みコーパスで9回使用の標準表現と同一用法）。効能は一人称体験（落ち着いた/変わった）と事実断定のみ
- **重複**: 既存128テーマ名と完全重複0。ボーナス既存4本との近似重複0
- **文字数**: 最長244字（Threads上限500に余裕）
- **ID衝突**: なし（新規201-218 / bonus 5-10）

## 5. 既存システムへの統合（※要承認）

`skin_threads_action.py` の読み込み仕様（`load_posts_for_variant` は id 一致で entry を返す / `pick_today_theme` は `len(threads_a)` をテーマ総数に使う）に適合済み。統合は以下の順で、**ユーザー承認後**に実施する。

1. **ボーナス弾の補充（低リスク・先行推奨）**: `skin_threads_bonus_posts.json` の `bonus_posts` に id 5-10 を追記。現在残弾0なので、反応good日の追撃が即復活する
2. **新規18本の投入（要判断）**: `skin_threads_posts.json` の `threads_a` へ 201-218 を追記すると、`theme_count` が 128→146 に増えて巡回に乗る。ただし **A/B測定（〜6/24）のテーマ番号がずれる**ため、測定終了後の投入を推奨
3. **将来のシステム改善（別タスク・診断⑤由来）**:
   - ハッシュタグ自動付与（`skin_hashtags.py` 3個ローテ）を1個へ、またはhuman_post本文内蔵に寄せる
   - CTAローテ（`skin_comment_seeder.py`）のフォロー誘導/リポスト懇願をエンゲージメントベイト回避のため見直し
   - human_postの1コメ目を汎用問いかけ固定でなく、投稿ごとの `seed_comment` フィールドで個別指定できるようにする
