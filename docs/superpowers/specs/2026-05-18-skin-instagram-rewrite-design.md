# skin Instagram カルーセル全面リライト 設計書

作成日: 2026-05-18
対象アカウント: skin（@skin_reset_jp / 肌荒れ改善Instagram）
目的: プロのコンテンツクリエイター視点でカルーセル投稿を全面刷新し、**保存率を旧版比+30%以上** に引き上げる

---

## 背景・現状

- skin Instagramはカルーセル形式で20テーマ（`skin/carousel_content.json`）を1日4回ローテーション投稿
- 投稿スクリプト: `skin/skin_instagram_auto_post.py`（GitHub Actions経由・watchdog付き）
- 進捗: `skin/skin_instagram_progress.json` の `next_index=5`（theme01〜05は旧版で投稿済み）
- 現状のスライドは5枚固定（cover→list/comparison×3→cta）で1枚あたりの情報量が多く、保存特性に最適化されていない
- キャプションは「保存してください」一辺倒、ハッシュタグも全テーマ共通9個で固定
- memoryに「現状PIL画像はNG認識、別サービスで再挑戦予定」と記載あり
- 直近の関連案件: skin Threads全面リライト（2026-05-17設計／2026-05-18 Batch1リリース済み）
  - フック3軸ローテーション・CTA4軸ローテーション・週次バッチの運用フレームを確立済み

## 設計セクション1/5: 全体アーキテクチャ

### スコープ対照表

| 項目 | 現状 | 新設計 |
|---|---|---|
| テーマ数 | 20テーマ | 20テーマ（維持） |
| スライド構成 | 5枚（cover→list/comp×3→cta） | **8枚（cover→hook→point×5→cta）** |
| 1スライドの情報量 | リスト2〜3項目／枚 | **1要点／枚** |
| キャプション | 固定文＋共通ハッシュタグ9個 | **4軸ローテーション＋テーマ別ハッシュタグ15個** |
| フック型 | 統一なし | **3軸ローテーション（数字/逆張り/エピソード）** |
| 画像生成 | 既存PIL（NG認識あり） | **PILテンプレを全面刷新（保存特化レイアウト）** |
| 投稿頻度 | 1日4回 | 1日4回（維持） |
| リリース | 進捗インデックス順 | **週次バッチ・4テーマ/週 × 5週間** |
| A/Bテスト | なし | なし（1案で磨く） |
| データ収集 | Threads APIのみ | **Instagram Graph API 対応を新規追加** |

### コア戦略（保存率最大化に集中）

1. **「あとで読み返したい」と思わせる8枚テンプレ**
   1スライド1要点で迷子にならない、目的別の章立てで読み返しやすい構成
2. **3軸フックローテーション**
   数字／逆張り／エピソードで毎回パターンを変え、フィードのスクロール疲労を防ぐ
3. **4軸CTAローテーション（Threadsと共通）**
   保存／コメント／note誘導／フォローを順番に出して飽きさせない、運用フレームをThreadsと共有
4. **週次バッチ4テーマでバージョン管理**
   旧テーマと新テーマが共存する期間を作り、保存率の差分でテンプレ自体を再検証
5. **PILテンプレ刷新**
   余白・タイポグラフィ・カラーテーマを保存特化レイアウトに置き換え

### 新旧並走の仕組み

- 各週、対象4テーマだけ `carousel_content.json` と画像を差し替え
- 旧16テーマは継続稼働、5週間で全20テーマが新設計に置き換わる
- 各バッチリリース前にGitタグを打つ（`skin-instagram-batch-N-before`）

## 設計セクション2/5: 8枚テンプレの中身

### スライド役割表

| # | 役割 | 1枚の主目的 | 文字量目安 |
|---|---|---|---|
| 1 | カバー | スクロールを止める（3軸フック） | 大見出し2行＋小見出し1行 |
| 2 | 共感フック | 「私のことだ」と読者を引き込む | 問題提起40〜80字 |
| 3 | ポイント① | 最重要1要点（一番効くこと） | 見出し1行＋本文4文 |
| 4 | ポイント② | 2番目に重要な1要点 | 同上 |
| 5 | ポイント③ | 3番目の1要点 | 同上 |
| 6 | ポイント④ | 4番目の1要点 | 同上 |
| 7 | ポイント⑤ | 締めの1要点（次の行動誘発） | 同上 |
| 8 | CTA | 4軸ローテーションのアクション喚起 | 強コピー1行＋補足1行 |

### フック3軸（スライド1）— ローテーション規則 `id % 3`

| 型 | 適用条件 | 例（スライド1のコピー） |
|---|---|---|
| **数字型** | `id % 3 == 1` | 「肌荒れを長引かせる<br>7つの習慣」／「腸活で本当に効いた<br>食べ物TOP5」 |
| **逆張り型** | `id % 3 == 2` | 「化粧水を変えても<br>肌は治りません」／「洗顔を減らしたら<br>肌が整った話」 |
| **エピソード型** | `id % 3 == 0` | 「8年間試した結果、<br>効いた習慣は3つだけ」／「鏡を見るのが嫌だった私が<br>3ヶ月で変わった話」 |

### スライド2（共感フック）

- 「あなたも当てはまるかも」の体裁で読者を引き込む
- スライド1の続きとして問題提起
- 例:「化粧水を3年変え続けても、肌は治りませんでした。原因は肌の外側ではなく、内側にありました」

### ポイントスライド（スライド3〜7）の4文構造

```
[見出し（1行）]
事実 → 具体例 → 数字 → 読み手にとっての意味
```

例（テーマ：腸内環境）:

```
見出し: ① 朝のみそ汁から始める

事実  : 腸内環境は肌の炎症と直結している。
具体例: みそ汁の麹菌と乳酸菌が善玉菌を増やす。
数字  : 1杯で約100億の乳酸菌を補える目安。
意味  : 内側からの改善は、朝1杯から始められる。
```

**薬機法ガード**:
- NG: 「効く」「治る」「○○%効果」「効果が出る」「絶対」
- OK: 「報告がある」「分かっている」「目安」「傾向」「変化がある」

### CTA 4軸（スライド8）— ローテーション規則 `id % 4`

| 型 | 適用条件 | 強コピー | 補足 |
|---|---|---|---|
| **保存** | `id % 4 == 1` | 保存して<br>読み返してね | 1つずつ試して<br>3ヶ月で変わる |
| **コメント** | `id % 4 == 2` | あなたが効いた<br>習慣を教えて | コメント欄で<br>みんなの実例を共有 |
| **note誘導** | `id % 4 == 3` | 改善の全工程は<br>noteにまとめた | プロフィールリンクから<br>すぐ読めます |
| **フォロー** | `id % 4 == 0` | 毎日肌改善の<br>ヒントを発信中 | フォローしてね |

### PILテンプレ刷新ポイント（`skin_instagram_image_gen_v4.py`）

- **キャンバス**: 1080×1080固定
- **余白**: 上下120px・左右80px（現状より広く取る）
- **タイポグラフィ**: NotoSerifJP（既存資産）／カバー大見出し72pt・本文32pt
- **配色**: テーマ別の `dark / accent / pale` 3色構成（既存`CARDS`定義を流用）
- **番号バッジ**: ポイントスライドに丸番号（①〜⑤）
- **フッターバー**: 全スライド共通の細いダーク帯＋ブランドコピー
- **行間**: 1.6倍（読み返し前提）

## 設計セクション3/5: キャプション・ハッシュタグ設計

### キャプションの統一フォーマット

```
[行1] フック再掲（スライド1のコピーと同じ）

[行2-4] 本文（カルーセルで何がわかるか予告／読みたくなる導入）

[行5-6] CTA行（4軸ローテーション）

#共通タグ #テーマ別タグ #サイズ別タグ
```

### 4軸CTA別キャプション雛形

**保存型 `id % 4 == 1`**
```
[フック再掲]

肌荒れが長引く人の共通点は、内側にあります。
8枚で「今日から変えられる習慣」をまとめました。

保存して、1つずつ試してみてください。
3ヶ月で肌が変わります。
```

**コメント型 `id % 4 == 2`**
```
[フック再掲]

あなたの「これが効いた」を教えてください。
コメント欄を読み返すと、実例の宝庫になっています。

あなたが効いた習慣を1つだけ、コメントで教えてください。
```

**note誘導型 `id % 4 == 3`**
```
[フック再掲]

カルーセル8枚で要点をまとめましたが、
詳しい改善ステップは note に全部書きました。

プロフィールのリンクから読めます。
8枚で「何を」がわかり、note で「どう進めるか」がわかります。
```

**フォロー型 `id % 4 == 0`**
```
[フック再掲]

このアカウントでは肌荒れを内側から治す習慣を毎日発信中。
8年間試してまとめた、本当に効いた習慣だけを投稿しています。

フォローして、明日の朝の投稿も一緒に。
```

### ハッシュタグ3層構造（計15個）

| 層 | 目的 | タグ数 | 例 |
|---|---|---|---|
| **共通固定** | アカウント識別 | 3個 | `#肌荒れ #大人ニキビ #肌改善` |
| **テーマ別** | 該当テーマの興味層に届ける | 7個 | 腸活: `#腸活 #発酵食品 #腸内環境 #みそ汁 #納豆 #食事改善 #美肌食` |
| **サイズ別ミックス** | 大中小タグで露出最大化 | 5個 | 大:`#美容` 中:`#美肌習慣` 小:`#肌の悩み相談` |

### ハッシュタグマッピング

`carousel_content.json` の各テーマに以下のキーを追加:

```json
{
  "id": 1,
  "fixed_hashtags": ["肌荒れ", "大人ニキビ", "肌改善"],
  "theme_hashtags": ["腸活", "発酵食品", "腸内環境", "食事改善", "美肌食", "みそ汁", "納豆"],
  "size_mix_hashtags": ["美容", "美肌習慣", "肌の悩み相談", "スキンケア初心者", "30代肌悩み"]
}
```

投稿スクリプトは3層を結合して末尾に付与する。

### 薬機法ガード（キャプションにも適用）

- NG: 「治る」「効く」「○○%効果」「絶対」
- OK: 「変わります」「分かっています」「目安」「報告がある」「傾向」
- 体験談は「私の場合」「個人の感想です」で逃げ道を作る

## 設計セクション5/5: 検証指標・ロールバック判定

### 主要KPI（毎日測定）

| 指標 | 計算式 | 目標値（旧版比） | 優先度 |
|---|---|---|---|
| **保存率** | saved / impressions | **+30%以上** | 最優先 |
| 保存数（絶対値） | 投稿あたりの保存数 | +30%以上 | 高 |
| 平均リーチ | impressions の30日平均 | +20%以上 | 中 |
| フォロワー増加/週 | follower_count の週次差分 | +10人以上 | 中 |
| note誘導クリック | プロフィールリンクのクリック数 | +20%以上 | 低 |

### データ収集（新規実装）

現状 `skin_data_collector.py` は Threads API のみで Instagram は未対応。Instagram Graph API 対応モジュールを **新規追加** する:

- 取得項目: `saved` / `impressions` / `reach` / `follower_count` / `profile_visits`
- 取得頻度: 毎日 06:00 JST（GitHub Actions で実行）
- 保存先: `skin/skin_instagram_analytics.json`（新規ファイル）

**ベースライン取得（重要）**: 初回起動時に Instagram Graph API から過去30日分の投稿データを遡って取得し、旧版（v1）の保存率ベースラインを確立する。これがないと Batch 1 の判定が機能しない。Batch 1 リリース前（2026-05-23まで）に最低14日分のv1データを蓄積することを必須条件とする。

### 検証レポート（新規）

`skin/skin_instagram_report.py` を作成し、バッチごとに2週間検証後にレポート出力:

```
=== Batch 1 検証レポート（2026-06-06）===

テーマ別 保存率（v2 vs v1平均）:
  テーマ1: v2=4.8%  v1平均=2.5%  → +92% 🟢
  テーマ2: v2=3.2%  v1平均=2.7%  → +18% 🟡
  テーマ3: v2=5.1%  v1平均=3.0%  → +70% 🟢
  テーマ4: v2=1.8%  v1平均=2.4%  → -25% 🔴

旧版比較:
  保存率: +39% ✓
  リーチ: +25% ✓
  フォロワー増加/週: +12人 ✓
  
判定: Batch 2 を続行（テーマ4のフックを再設計）
```

### ロールバック判定基準（3段階）

| レベル | 条件 | アクション |
|---|---|---|
| **黄信号** | 保存率が旧版ベースライン-10%以上下落（3日連続） | 該当テーマのコピー差し替えを準備、Batch停止 |
| **赤信号** | 保存率が旧版ベースライン-30%以上下落（2日連続） | バッチ全体をロールバック（5分手順） |
| **緊急** | 警告コメント／薬機法指摘／凍結リスク兆候 | 即座に全バッチロールバック＋投稿停止 |

**Batch 1 の特別運用**: 旧版ベースラインが14日分未満の場合、上記比較判定は機能しない。代替として絶対値で判定する:
- 黄信号: 新版テーマの保存率が **1.5% 未満**（3日連続）
- 赤信号: 新版テーマの保存率が **0.8% 未満**（2日連続）

Batch 2 以降は旧版ベースラインが十分に蓄積されているため、通常の相対判定に切り替える。

### 学習サイクル

```
Batch 1 検証完了
    ↓
保存率の高い「フック型・本文構造・CTA軸」を特定
    ↓
Batch 2 制作に反映（勝ちパターンの比率を上げる、負けパターンは外す）
    ↓
... Batch 5完走時には「保存される20テーマ」が完成
```

### 成功基準（プロジェクト完了の定義）

Batch 5検証終了時点（2026-07-04）で以下を満たす:

- 全20テーマの **保存率が旧版比+30%以上**
- フォロワー増加が **週+10人以上** を継続
- **薬機法・凍結リスク兆候なし** で稼働継続
- 「保存→再表示→フォロー」のループが `skin_instagram_analytics.json` で観測できる

→ 達成後の展開: 21〜30テーマの新規追加、Higgsfield AIによるカバービジュアル化、ronin Instagramへの同フレーム展開を検討。
