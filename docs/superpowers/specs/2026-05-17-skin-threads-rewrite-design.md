## 設計セクション2/5: テンプレート集

### 本1（フック）3型ローテーション

ローテーション規則: `id % 3`

| 型 | 適用条件 | 特徴 | 例 |
|---|---|---|---|
| **数字フック** | id%3==1 | 期間・回数・結果を数字で提示 | 「8年間試した結果、効いたのはたった3つだった」 |
| **逆張り** | id%3==2 | 常識を否定する切り口 | 「化粧水を変えても肌は治りません」 |
| **エピソード** | id%3==0 | 個人体験から入る | 「鏡を見るのが嫌だった私が、3ヶ月で変わった話」 |

各型につき4実例以上をテンプレ化し、テーマに応じてアレンジ。

### 本2（本題）統一フォーマット

**4文構造**: 事実 → 具体例 → 数字 → 読み手にとっての意味

例（腸内環境テーマ）:
```
腸の状態は肌に直接影響することが分かっています。
ヨーグルトや味噌などの発酵食品を毎日摂る習慣が腸内環境を整えます。
研究では腸内細菌のバランスが整っている人ほど肌トラブルが少ない傾向が報告されています。
つまり外側のケアより先に、内側を整えることで肌が変わる可能性があります。
```

**数字使用方針（確定）**:
- 数字を**使う**（薬機法注意）
- 断定NG表現:「効く」「治る」「○○%効果」
- OK表現:「報告がある」「分かっている」「目安」「傾向」

### 本3（CTA）4型ローテーション

ローテーション規則: `id % 4`

| 型 | 適用条件 | 用途 | 例 |
|---|---|---|---|
| **保存** | id%4==1 | リーチ拡大 | 「読み返したい人は保存してね」 |
| **コメント** | id%4==2 | エンゲージメント | 「あなたが試して効いた習慣を教えて」 |
| **note誘導** | id%4==3 | マネタイズ | 「詳しい改善法はnoteにまとめた→[link]」 |
| **フォロー** | id%4==0 | アカウント成長 | 「毎日肌改善のヒントを発信中。フォローしてね」 |

### ハッシュタグ

本3のみに3〜5個付与。
- 基本セット: `#肌荒れ #大人ニキビ #肌改善`
- テーマ別追加: `#腸活` `#睡眠` `#食事` 等

## 設計セクション3/5: A/Bテスト機構

### 216セット構造

```
skin_threads_posts.json
├─ threads_a: [108セット]  ← A系統
└─ threads_b: [108セット]  ← B系統
```

### A/Bの違い設計（本題切り口で差別化）

| 軸 | Aパターン | Bパターン |
|---|---|---|
| フック型 | 3型ローテーション（数字/逆張り/エピソード） | 3型ローテーション（数字/逆張り/エピソード） |
| 本題切り口 | **原因解説**（なぜ起きるか） | **解決策提示**（どうすればいいか） |
| CTA型 | 4型ローテーション（保存/コメント/note誘導/フォロー） | 4型ローテーション（保存/コメント/note誘導/フォロー） |

→ フック・CTAは共通のローテーションを使い、**本題切り口のみで差別化**。検証軸を1つに絞ることで「原因解説 vs 解決策提示、どちらが伸びるか」がクリーンに測れる。

### ローテーション管理（skin_threads_progress.json 拡張）

```json
{
  "daily_index": 0,
  "ab_index_per_theme": {
    "1": "A",
    "2": "B",
    ...
  },
  "ab_results": {
    "1": {"a_score": 0, "b_score": 0, "a_posts": 0, "b_posts": 0},
    ...
  }
}
```

**動作**:
1. 投稿時、daily_indexで今日のテーマ番号を決定
2. ab_index_per_theme[テーマ番号] を見てAかBを選択
3. 投稿後、選んだ方を反転（A→B、B→A）

### 勝者判定ロジック

- **engagement_score** = いいね×1.0 + コメント×2.5 + シェア×5.0
- バッチ1リリースから2週間後に判定
- A/B両方で**最低3回ずつ投稿済み**のテーマのみ判定対象
- A平均 vs B平均で**20%以上差**があれば勝者確定
- 20%以下なら両方継続（差がない＝どちらも有効）

### 勝者確定後の運用

| 判定結果 | 運用 |
|---|---|
| Aが勝者 | Aのみ投稿（Bは凍結） |
| Bが勝者 | Bのみ投稿（Aは凍結） |
| 差がない | A/B両方継続 |
| 両方低い | C版を新規制作 |

## 設計セクション5/5: 検証指標・ロールバック

### 主要KPI（毎日測定）

| 指標 | 計算式 | 目標値（旧版比） |
|---|---|---|
| engagement_score | いいね×1.0 + コメント×2.5 + シェア×5.0 | +20%以上 |
| 平均リーチ | impressions の30日平均 | +10%以上 |
| コメント率 | comments / impressions | +50%以上 |
| 保存率 | saved / impressions | +30%以上 |
| フォロワー増加数/週 | follower_count の週次差分 | +5人以上/週 |

### データ収集（既存稼働中・変更なし）

- `skin_data_collector.py`: 毎日06:00 JSTにThreads APIから取得
- `skin_analyzer.py`: 毎日07:00 JSTに分析実行

### 検証レポート（新規スクリプト）

`skin/skin_ab_report.py` を作成し、バッチごとに2週間検証後にレポート出力:

```
=== Batch 1 検証レポート（2026-06-06）===

テーマ別A/B結果:
  テーマ1: A=85.3, B=62.1 → A勝ち（37%差）
  テーマ2: A=70.2, B=78.5 → 差なし（11%）→両方継続
  ...

全体サマリ:
  A勝ち: 8テーマ / B勝ち: 5テーマ / 差なし: 7テーマ
  
旧版比較:
  engagement_score: +24% ✓
  リーチ: +15% ✓
  コメント率: +62% ✓
  保存率: +41% ✓
  フォロワー増加/週: +8人 ✓
```

### ロールバック判断基準（3段階）

| レベル | 条件 | アクション |
|---|---|---|
| 黄信号 | engagement_score が**旧版-10%以上下落**（3日連続） | Batch停止、原因分析開始 |
| 赤信号 | engagement_score が**旧版-30%以上下落**（2日連続） | バッチ全体をロールバック |
| 緊急 | 警告コメント、薬機法指摘、凍結リスク兆候 | 即座に全バッチロールバック |

### ロールバック手順

```bash
# Step 1: Gitタグから復元
git checkout skin-threads-batch-N-before -- skin/skin_threads_posts.json
git checkout skin-threads-batch-N-before -- skin/skin_threads_progress.json
git checkout skin-threads-batch-N-before -- .github/workflows/skin_threads_post.yml

# Step 2: コミット&プッシュ
git commit -m "Rollback batch N"
git push
```

所要時間: 5分程度

### 学習サイクル

```
Batch 1 検証完了
    ↓
勝ちパターン分析
    ↓
Batch 2 制作に反映（勝ちパターンの比率を上げる）
    ↓
... 6バッチ完走時には最強の216セットが完成
```

### 成功基準（プロジェクト完了の定義）

Batch 6検証終了時点（2026-07-11）で:
- 216セット中、勝者または「差なし継続」判定が**70%以上**
- engagement_score が旧版比**+20%以上**維持
- フォロワー増加が**週+5人以上**達成
- 薬機法・凍結リスクなく稼働継続

→ 達成後、@RoninWordsアカウントにも同じ仕組みを展開検討
