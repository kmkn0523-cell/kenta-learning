プライバシーポリシー

最終更新日: 2026年6月20日

本ポリシーは、スキンケア／インナーケア領域のコンテンツ運用を支援するツール（以下「本ツール」、Threadsアカウント @skin_reset_jp の運用に使用）が、Threads API を通じて取り扱う情報について定めるものです。

1. 運営者・連絡先

運営者: skin_reset_jp 運用者（個人）
お問い合わせ: kmkn0523@gmail.com

2. 取得する情報

本ツールは Threads API の keyword_search を用いて、特定のキーワード（例:「腸活」「肌荒れ」「インナーケア」）に関連する公開投稿を取得します。取得するのは、投稿本文・投稿者の公開ユーザー名・投稿日時・投稿IDなど、Threadsが公開している情報に限られます。非公開情報・ダイレクトメッセージ・連絡先・位置情報は取得しません。

3. 利用目的

取得した公開投稿は、スキンケア／インナーケアに関連する話題のモニタリングと、関連性を確認した上での有益な情報提供を目的とした返信にのみ使用します。無差別な送信や広告目的の利用は行いません。

4. 送信における配慮（スパム防止）

無差別な送信を防ぐため、1日あたりの返信上限・同一ユーザーへの上限・送信間隔のランダム化・定型文の連続使用禁止を実装しています。

5. 第三者提供

取得した情報を第三者に販売・提供することはありません。

6. データの保持

重複返信を防ぐ目的で、返信済み投稿のIDのみを記録します。それ以外の投稿内容を恒久的に保存することはありません。アクセストークンは GitHub Secrets 等の安全な場所で管理し、公開しません。

7. ユーザーの権利

本ツールによる返信の削除や、情報の取り扱いに関するお問い合わせは、上記の連絡先までご連絡ください。

8. 改定

本ポリシーは必要に応じて改定されることがあります。改定時は本ページの最終更新日を更新します。

Privacy Policy (English)

Last updated: June 20, 2026

This policy describes how a content-operations tool (the "Tool", used to operate the Threads account @skin_reset_jp in the skincare / inner-care space) handles information accessed through the Threads API.

1. Operator & Contact

Operator: operator of skin_reset_jp (individual)
Contact: kmkn0523@gmail.com

2. Information We Access

The Tool uses the Threads API keyword_search endpoint to retrieve public posts related to specific keywords (e.g., gut health, skin troubles, inner care). It accesses only publicly available information such as post text, the author's public username, timestamp, and post ID. It does not access private information, direct messages, contacts, or location data.

3. Purpose of Use

Retrieved public posts are used only for monitoring topics related to skincare / inner-care and for sending replies that provide helpful information after a relevance check. The Tool is not used for indiscriminate sending or advertising.

4. Anti-Spam Measures

To prevent indiscriminate sending, the Tool enforces a daily reply cap, a per-author cap, randomized send intervals, and a no-repeat rule for templated text.

5. No Third-Party Sharing

We do not sell or share the retrieved information with third parties.

6. Data Retention

To prevent duplicate replies, only the IDs of posts already replied to are stored. No other post content is retained permanently. Access tokens are stored securely (e.g., GitHub Secrets) and are never published.

7. Your Rights

For removal of replies or questions about data handling, please contact us at the address above.

8. Changes

This policy may be updated as needed; the "Last updated" date will reflect any changes.
