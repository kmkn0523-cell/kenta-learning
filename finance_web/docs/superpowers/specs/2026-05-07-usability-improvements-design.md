# 家計管理アプリ ユーザビリティ改善 設計書

作成日: 2026-05-07  
対象ファイル: `finance_web/src/App.jsx`（1ファイルで完結）

---

## 概要

8つの改善を App.jsx に加える。  
データ構造・localStorage のキー・外部依存は変えない。UI と入力処理の改善のみ。

---

## 変更詳細

### 1. 固定費の初期データをゼロにする

**現状:** `INIT_FX` に家賃¥80,000・電気¥8,000・水道¥3,000・スマホ¥5,000・光回線¥4,000 が定義されており、初回起動からダッシュボードの数字が狂う。

**変更:** `INIT_FX` を空配列 `[]` にする。

```js
// 変更前
const INIT_FX = [ {id:1,name:"家賃",...}, ... ];

// 変更後
const INIT_FX = [];
```

固定費タブの空の状態ガイダンス（改善8）でフォローする。

---

### 2. 金額入力のカンマ対応

**現状:** `Number("1,000,000")` は `NaN` になるため、大きな金額を入力するとき「1000000」と打たなければならない。

**変更:** `parseAmount` ヘルパー関数を追加し、全ての `Number(...)` による金額変換をこれに置き換える。

```js
// カンマを除去してから数値に変換するヘルパー
function parseAmount(v) {
  return Number(String(v).replace(/,/g, ""));
}
```

対象箇所: `addTx`, `addInc`, `addFx`, `LoanSection.save`, `LoanSection.pay`, 口座残高の保存処理。

---

### 3. 口座残高をダッシュボードに移動

**現状:** 口座残高カードが「収入タブ」上部にあり、収入入力の邪魔になっている。

**変更:**
- 収入タブ（`tab==="inc"`）から口座残高カードのJSXを削除
- ダッシュボード（`tab==="dash"`）の収入合計カードのすぐ下に移動
- 収入タブには口座残高の編集フォームも不要になるため削除し、ダッシュボードに統合

口座残高カード自体のUI（表示・追加・編集・削除）はそのまま維持する。

---

### 4. 支出フォームを折りたたみ式に変更

**現状:** 支出タブを開くと常にフォームが展開されており、過去の支出リストを見るのに邪魔。

**変更:**
- デフォルトは `showTxForm = false`（非表示）
- 「＋ 支出を追加」ボタンを設置。タップでフォームを展開
- 追加ボタンを押した後も**フォームは開いたまま**（連続入力しやすい）
- フォームが開いているときはボタンラベルを「✕ 閉じる」に変更

```js
const [showTxForm, setShowTxForm] = useState(false);
```

---

### 5. ローンタブの分類ラベルに補足を追加

**現状:** タブラベルが「🏧 消費者金融」「🏛 銀行/リボ」「🏦 その他」のみで、違いが分かりにくい。

**変更:** タブラベルに括弧で例を追記する。

```js
// 変更前
[["cf","🏧 消費者金融"],["bl","🏛 銀行/リボ"],["lo","🏦 その他"]]

// 変更後
[["cf","🏧 消費者金融（アコム・プロミス等）"],["bl","🏛 銀行・リボ払い"],["lo","🏦 住宅・車ローン等"]]
```

---

### 6. 収入タブにCSV出力ボタンを追加

**現状:** CSV出力は支出タブのみ。収入データはエクスポートできない。

**変更:** 収入タブのリスト上部に「📥 CSV出力」ボタンを追加。  
`exportIncCSV` 関数を新規追加（`exportCSV` と同じ構造で収入データ版）。

```js
function exportIncCSV() {
  const rows = [["日付","カテゴリ","金額","メモ"],
    ...[...mInc].sort((a,b)=>b.date.localeCompare(a.date))
      .map(i=>[i.date, i.category, i.amount, i.note||""])];
  // ... CSVダウンロード処理（exportCSV と同じ）
}
```

---

### 7. 収入タブにカテゴリフィルタ＋テキスト検索を追加

**現状:** 収入リストに検索・フィルタ機能がない。

**変更:** 支出タブと同じUIを収入タブにも追加。
- テキスト検索 state: `incSq`
- カテゴリフィルタ state: `incFcat`
- フィルタ済みリスト: `fInc`（`useMemo` で計算）

```js
const [incSq, setIncSq] = useState("");
const [incFcat, setIncFcat] = useState("");
const fInc = useMemo(() => {
  let l = mInc;
  if (incFcat) l = l.filter(i => i.category === incFcat);
  if (incSq.trim()) {
    const q = incSq.toLowerCase();
    l = l.filter(i => (i.note||"").toLowerCase().includes(q) || String(i.amount).includes(q));
  }
  return l;
}, [mInc, incFcat, incSq]);
```

---

### 8. 空の状態にガイダンスを追加

**現状:** 「データがありません」と表示されるだけで、次にすべきことが分からない。

**変更:** 各タブの空表示に一言ガイダンスを追加する。

| タブ | 空ガイダンス |
|------|-------------|
| 収入 | 「今月の収入を上のフォームから入力しましょう」 |
| 支出 | 「今月の支出を「＋ 支出を追加」から記録しましょう」 |
| 固定費 | 「家賃・サブスクなど毎月固定でかかる費用を登録しましょう」 |
| ダッシュボード（口座） | 「口座を追加すると残高合計が確認できます」 |

---

## 変更しないもの

- データ構造（localStorage のキー名・JSON の形）
- パスワード認証の仕組み
- ローン計算ロジック（`tInt`, `mInt`, `cDate`）
- ダッシュボードの収支バランスバー・ローン残債一覧
- GitHub Actions / Vercel デプロイ設定

## テスト観点

1. 初回アクセス時に固定費が空になっていること
2. 金額欄に「1,000,000」と入力してもエラーにならず正しく保存されること
3. ダッシュボードに口座残高カードが表示され、収入タブには表示されないこと
4. 支出タブで「＋ 支出を追加」をタップするとフォームが展開し、追加後も開いたままになること
5. 収入タブでカテゴリフィルタ・テキスト検索が機能すること
6. 収入タブのCSVが正しくダウンロードされること
7. ローンタブのサブタブラベルに補足が表示されること
8. 各タブの空の状態にガイダンス文が表示されること
