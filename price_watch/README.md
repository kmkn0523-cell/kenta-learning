# iPhone買取価格チェッカー

りんごの奴隷（買取サイト）の **iPhone 17 Pro Max 512GB シルバー** の買取価格を自動で見に行き、
変化があればGmailで自分宛にメールで知らせてくれるプログラムです。
毎回サイトを開いて価格を確認する手間を、プログラムが代わりにやってくれます。

---

## 監視している対象

| 項目 | 値 |
|------|----|
| 機種 | iPhone 17 Pro Max |
| 容量 | 512GB |
| 色 | シルバー（色で価格が違うので1色に限定）|

この3つ**すべてに一致する行だけ**を価格表から拾います。色違い（ディープブルー等）は無視します。

---

## 全体の流れ（6ステップ）

```
①サイトにログイン → ②価格ページを開く → ③シルバーの価格を読み取る
   → ④前回の値と比べる → ⑤メールで知らせる → ⑥今回の値を記録する
```

### ① サイトにログインする
`Playwright`（画面が見えない自動ブラウザ）が、人間の代わりにメール・パスワードを入力してログインします。
メールやパスワードはコードに直接書かず、`.env` ファイルやGitHubのSecretsから読み込みます（盗まれないための工夫）。

### ② 価格ページを開いて表を読み取る
価格ツールのページを開き、表の各行（`tr`）から「機種名」と「行のテキスト全体」を取り出します。

### ③ シルバーの価格だけを読み取る
`parse_prices` 関数が、機種・容量・色の3つすべてに合う行だけから価格を拾います。

価格の見分け方:
- **店舗の買取価格** = 「`230,000 円`」のように数字と「円」の間に**スペースがある**もの
- **定価** = 「`¥229,800`」のように**¥マーク付き**のもの

拾った価格から計算するもの:
- **最高価格** = 一番高い店舗の値
- **平均価格** = 全店舗の値を足して店舗数で割った値（四捨五入）

### ④ 前回の値と比べる
記録ファイル `iphone_price_state.json` から前回の最高価格を読み、今回と比べます。
- **初めての実行**（記録なし）→ メールは送らず記録だけ
- **最高価格が前回と違う** → 「値動きあり」としてメール
- **最高価格が前回と同じ** → 「変化なし」でも念のため毎回メール

### ⑤ メールで知らせる
Gmailで自分宛にメールを送ります。件名・本文に入る情報:
- 最高価格と、定価から±いくらか
- 平均価格と、定価から±いくらか

特別な表示:
- 買取が**定価ピッタリ** → 「定価と同額」
- 買取が**定価を超えた**（通常は起きない想定）→ 件名の先頭に **「【事件】」**

### ⑥ 今回の値を記録する
今回の最高価格・平均価格などを `iphone_price_state.json` に保存します。
これが**次回の「前回の値」**になります。

---

## メール件名・本文の例

**普段（定価より安いとき）**
```
【変化なし】17ProMax512GB銀 最高229,500円(定価-300) 平均228,200円(定価-1,600)
```

**値動きがあったとき**
```
【値動き+500円】17ProMax512GB銀 最高230,000円(定価-… ) 平均…
```

**買取が定価ピッタリのとき**
```
【変化なし】17ProMax512GB銀 最高229,800円(定価ぴったり) 平均…
```

**買取が定価を超えたとき（異常）**
```
【事件】【値動き+500円】17ProMax512GB銀 最高230,300円(定価+500) 平均…
```

本文の例:
```
最高 229,800円（定価 ¥229,800 と同額）
平均 228,500円（定価 ¥229,800 から -1,300円）
```

## 安全のための工夫

- 価格取得に失敗したら（サイトの作りが変わった・ログイン失敗等）、黙って終わらず
  **「【取得失敗】」** のメールを送って異常を知らせます。
- 定価が取得できなかった場合は「【事件】」を誤って付けないようにしています。

## 自動で動く仕組み

`GitHub Actions` という仕組みで決まった時刻に自動実行されます（1日3回）。
何もしなくても勝手にチェックしてメールが届きます。
ワークフロー定義: `.github/workflows/iphone_price_watch.yml`

## ファイル構成

| `iphone_price_watcher.py` | 本体（ログイン〜価格取得〜メール送信〜記録）|
| `iphone_price_state.json` | 前回価格の記録（プログラムが自動で書き換える。手動編集しない）|

## 必要な環境変数（秘密情報）

| 変数名 | 用途 |
|--------|------|
| `RINGONODOREI_EMAIL` | りんごの奴隷のログイン用メール |
| `RINGONODOREI_PASSWORD` | りんごの奴隷のログイン用パスワード |
| `GMAIL_ADDRESS` | 通知メールの送信元Gmail |
| `GMAIL_APP_PASSWORD` | Gmailのアプリパスワード |
| `NOTIFY_TO` | 通知先（省略時は送信元と同じ）|

ローカルでは `.env`、GitHub Actionsでは Secrets から読み込みます。

---

## 主な関数

| 関数 | やること |
|------|----------|
| `parse_prices` | 1行から店舗買取価格をすべてリストで取り出す（対象外は空リスト）|
| `parse_max_price` | 1行から最高買取価格を取り出す |
| `parse_list_price` | 1行から定価（¥付き）を取り出す |
| `fetch_highest_price` | ログイン〜価格ページ取得〜最高値・平均を返す |
| `format_vs_list_price` | 定価からの差を詳しい文章にする（本文用）|
| `short_vs_list` | 定価からの差を短く表す（件名用）|
| `save_price` | 今回の値を記録ファイルに保存 |
| `send_email` | Gmailで通知メールを送る |
| `main` | 全体の流れを実行 |
