# skin Threads ツリー投稿 設計書

作成日: 2026-05-04

---

## 概要

skinアカウントのThreads自動投稿を、単体投稿からツリー構造（スレッド形式）に変更する。
1回の投稿発火で3本のつながった投稿（フック→本題→CTA）を自動で連鎖投稿する。

---

## 要件

- 構成: 3本構成（フック → 本題 → CTA）
- セット数: 14セット（2週間分ローテーション）
- コンテンツ: Claudeが新規生成（既存の `skin_x_posts.json` は使わない）
- 自動投稿: 既存のGitHub Actions（1日4回）をそのまま利用

---

## データ設計

### 新ファイル: `skin/skin_threads_posts.json`

```json
{
  "threads": [
    {
      "id": 1,
      "theme": "テーマ名",
      "posts": [
        "本1（フック）: 500文字以内",
        "本2（本題）: 500文字以内",
        "本3（CTA）: 500文字以内"
      ]
    }
  ]
}
```

- 各 `posts` 配列は必ず3要素
- 各要素は500文字以内（Threads APIの上限）
- ハッシュタグ（`#肌荒れ #大人ニキビ` 等）は3本目にのみ付与
- `skin_x_posts.json` は変更しない

### 進捗管理: `skin/skin_threads_progress.json`

既存ファイルをそのまま継続使用。`daily_index` が0〜13をローテーション。

---

## スクリプト設計

### 変更対象: `skin/skin_threads_action.py`

#### 変更点

1. **読み込みファイルを変更**
   - 旧: `skin_x_posts.json`
   - 新: `skin_threads_posts.json`

2. **`post_to_threads()` 関数に `reply_to_id` 引数を追加**
   - `reply_to_id=None` のとき: 通常の新規投稿
   - `reply_to_id=<ID>` のとき: 指定IDへのリプライ投稿

3. **3本連鎖投稿の処理を追加**

#### 投稿フロー（1回の実行）

```
1. skin_threads_posts.json から該当セットを読み込む
2. 本1（フック）を投稿 → 30秒待機 → 公開 → post_id_1 を取得
3. 本2（本題）を post_id_1 にリプライ投稿 → 30秒待機 → 公開 → post_id_2 を取得
4. 本3（CTA）を post_id_2 にリプライ投稿 → 30秒待機 → 公開
5. daily_index を +1 して保存（14セットでループ）
```

合計所要時間: 約90〜120秒（GitHub Actions タイムアウト上限6時間に対して問題なし）

#### Threads API リプライ投稿の仕様

```
POST https://graph.threads.net/{USER_ID}/threads
パラメータ:
  media_type: TEXT
  text: 投稿本文
  reply_to_id: リプライ先の投稿ID  ← これを追加
  access_token: ...
```

## 変更しないもの

- `skin_x_posts.json`（既存投稿データ）
- `skin_threads_auto_post.py`（手動投稿スクリプト）
- GitHub Actionsワークフロー（発火タイミング・頻度は変えない）
- `.env` ファイル・APIキー

---

## エラーハンドリング

- 本1の投稿失敗時: スクリプトを即停止（本2・3を投稿しない）
- 本2・3の投稿失敗時: エラーログを出力して停止
- いずれの場合も `daily_index` は進めない（同じセットをリトライ可能にする）
